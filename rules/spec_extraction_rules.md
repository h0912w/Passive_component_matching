# 스펙 추출 규칙 문서
**경로**: `/rules/spec_extraction_rules.md`
**역할**: 규칙 기반 파서의 모든 처리 패턴을 정의하는 살아있는 문서.
새 패턴 발견 시 파서 코드보다 이 문서를 먼저 갱신한다.

---

## §1. 저항값 표기 패턴

### 1.1 R-표기법 (IEC 60062)
```
0R    → 0 Ω         0R0   → 0 Ω
1R    → 1 Ω         1R0   → 1 Ω
4R7   → 4.7 Ω       1R5   → 1.5 Ω
10R   → 10 Ω        100R  → 100 Ω
330R  → 330 Ω
```

### 1.2 K-표기법
```
1K    → 1,000 Ω     1K0   → 1,000 Ω
2K2   → 2,200 Ω     4K7   → 4,700 Ω
10K   → 10,000 Ω    100K  → 100,000 Ω
```

### 1.3 M-표기법
```
1M    → 1,000,000 Ω    2M2   → 2,200,000 Ω
4M7   → 4,700,000 Ω    10M   → 10,000,000 Ω
```

### 1.4 소수점 표기 (단위 추론 필요)
```
4.7       → 단위 불명확, 후속 토큰으로 추론
4.7ohm    → 4.7 Ω
4.7Ω      → 4.7 Ω
4.7R      → 4.7 Ω
4.7k      → 4,700 Ω
4.7K      → 4,700 Ω
4.7kohm   → 4,700 Ω
4.7M      → 4,700,000 Ω
```

### 1.5 단위 표기 변형 (모두 동일하게 처리)
```
ohm, Ohm, OHM, Ω, ω (소문자 오메가)
k, K, kΩ, kOhm, KΩ
m, M, MΩ, MOhm (주의: m과 M은 다른 단위 — m=밀리, M=메가)
```

### 1.6 파서 처리 순서
1. `[숫자][R/K/M][숫자]` 패턴 우선 매칭 (IEC 표기)
2. `[숫자][단위문자열]` 패턴 매칭
3. 숫자만 남을 경우 후속 토큰으로 단위 추론
4. 추론 불가 시 GLM 보조 요청

## §2. 패키지 표기 패턴

### 2.1 메트릭 ↔ 인치 양방향 alias 매핑
| 메트릭 | 인치 | 크기 (mm) |
|---|---|---|
| 0402 | 01005 | 0.4 × 0.2 |
| 0603 | 0201 | 0.6 × 0.3 |
| 1005 | 0402 | 1.0 × 0.5 |
| 1608 | 0603 | 1.6 × 0.8 |
| 2012 | 0805 | 2.0 × 1.2 |
| 3216 | 1206 | 3.2 × 1.6 |
| 3225 | 1210 | 3.2 × 2.5 |
| 5025 | 2010 | 5.0 × 2.5 |
| 6332 | 2512 | 6.3 × 3.2 |

> ⚠️ `0402`는 메트릭(0.4×0.2mm)과 인치(1.0×0.5mm) 두 의미가 있다.
> 컨텍스트 없이 단독 입력 시 **인치 0402(=메트릭 1005)**로 해석한다.
> (현장에서 "0402"는 대부분 인치 기준)

### 2.2 표기 변형
```
1608, 1608M, 0603, 0603I, 0603 Inch, 0603inch
"0603 (1608 Metric)", "1608/0603"
```

### 2.3 특수 패키지
```
2512 → 대전력용, 인치 기준
MELF, DO-213 → 원통형, 파서에서 플래그 처리
```

## §3. 오차 표기 패턴

### 3.1 % 표기
```
1%,  5%,  10%,  0.1%,  0.5%
±1%, ±5%, ±10%
+/-1%, +/-5%
```

### 3.2 문자코드 (IEC 60062)
```
B → ±0.1%
C → ±0.25%
D → ±0.5%
F → ±1%      ← 초기 지원
G → ±2%
J → ±5%      ← 초기 지원
K → ±10%
M → ±20%
```
> 초기 지원: F, J만 파서 직접 처리. 나머지는 GLM 보조 경로.

### 3.3 파서 처리 주의사항
- `M`은 저항값 단위(메가)와 오차 코드(±20%) 중의성 존재
  → 앞에 숫자가 붙은 경우(`2M2`, `1M`)는 저항값으로 해석
  → 단독으로 나타나는 경우(`M`)는 오차 코드로 해석
- `K`는 저항값 단위(킬로)와 오차 코드(±10%) 중의성 존재
  → 동일한 규칙 적용

## §4. 전력 표기 패턴

### 4.1 분수 표기
```
1/20W → 0.050W     1/16W → 0.0625W
1/10W → 0.100W     1/8W  → 0.125W
1/6W  → 0.167W     1/5W  → 0.200W
1/4W  → 0.250W     1/3W  → 0.333W
1/2W  → 0.500W     1W    → 1.000W
2W    → 2.000W
```

### 4.2 소수점 표기
```
0.05W, 0.0625W, 0.063W (≈ 1/16W)
0.1W, 0.125W, 0.25W, 0.5W, 1W, 2W
```

### 4.3 mW 표기
```
50mW → 0.05W    62.5mW → 0.0625W
100mW → 0.1W    125mW → 0.125W
250mW → 0.25W   500mW → 0.5W
1000mW → 1W
```

### 4.4 단위 변형
```
W, w, Watt, watt, WATT
mW, mw, milliwatt
```

## §5. 구분자 변형 처리

### 5.1 전처리 파이프라인
```
원본 입력
  → 1단계: 연속 공백 정규화 (탭, 다중 공백 → 단일 공백)
  → 2단계: 구분자 통일 (/, _, -, ;, , → 공백)
  → 3단계: 괄호 제거 또는 공백으로 치환
  → 4단계: 대소문자 처리 (저항값 단위는 대소문자 구분, 나머지 무시)
  → 토큰화
```

### 5.2 구분자 유형별 처리 규칙
| 구분자 | 처리 방법 |
|---|---|
| 공백 ` ` | 그대로 토큰 경계 |
| 슬러시 `/` | 분수 전력값(`1/8W`)이 아닌 경우 공백으로 치환 |
| 언더바 `_` | 공백으로 치환 |
| 대시 `-` | 음수가 아닌 경우 공백으로 치환 |
| 콤마 `,` | 공백으로 치환 |
| 세미콜론 `;` | 공백으로 치환 |
| 탭 `\t` | 공백으로 치환 |
| 괄호 `()[]` | 제거 후 내부 내용을 토큰으로 처리 |

### 5.3 슬러시 중의성 해결
```
1/8W  → 전력 분수 표기로 보존 (숫자/숫자W 패턴)
1/0603 → 구분자로 처리 (1, 0603 분리)
```

### 5.4 구분자 없는 연결 패턴 (어려운 케이스)
```
"1k0603"  → "1k" + "0603" (저항값 + 패키지 경계 추론)
"0R0805"  → "0R" + "0805"
```
> 이 패턴은 신뢰도 점수가 낮게 계산되어 GLM 더블체크 투입 대상이 된다.

## §6. 파서 신뢰도 점수 계산

### 6.1 계산식 (조합 방식)
```
신뢰도 = 필드 완성도(60%) + 토큰 매칭률(40%)
```

### 6.2 필드 완성도 (60% 가중치)
```
resistance 추출 성공: +40pt
package 추출 성공:    +20pt
tolerance 추출 성공:  +20pt
power 추출 성공:      +20pt
→ 100pt 만점 → 0.0~1.0 정규화
```

### 6.3 토큰 매칭률 (40% 가중치)
```
입력 토큰 수 대비 규칙 패턴에 매칭된 토큰 비율
예: "1k 1608 5% 0.25W" → 4/4 = 1.0
예: "1k ???? 5%"       → 2/3 = 0.67
```

### 6.4 GLM 투입 기준
```
total < 0.70              → GLM 보조 추출 투입
0.70 ≤ total < 0.85
  + 특이 패턴 감지        → GLM 더블체크 투입
total ≥ 0.85              → 코드 결과 확정
```

### 6.5 Mouser DB 교차 확인 보너스
- 추출된 값이 `/db/mouser_resistor_specs.json` DB에 실제 존재하면 신뢰도 +5pt 추가
