# Test Cases Generator Agent

에이전트: 랜덤 저항값 테스트 케이스 생성기

## 역할

테스트 실행 전 GLM API를 사용하여 다양한 랜덤 저항값 테스트 케이스를 생성하고 문서로 저장하는 에이전트입니다.

## 사용법

```
/test-cases-generator
```

## 작업 흐름

1. GLM API 호출하여 랜덤 저항값 테스트 케이스 생성
2. 생성된 케이스를 `tests/test-cases.json`에 저장
3. 테스트 러너는 이 파일을 읽어서 테스트 실행

## 생성할 테스트 케이스

각 카테고리별 5개씩, 총 20개:

### 1. 구조적 표준 입력 (Structured)
- 공백/슬래시/언더바 구분자
- 토큰 순서 랜덤 (저항값이 맨 앞이 아닐 수 있음)
- 저항값: 1k, 4.7k, 10K, 2.2M, 100R, 4R7, 1k5, 2R2
- 패키지: 0201, 0402, 0603, 0805, 1206
- 오차: 0.1%, 1%, 5%

### 2. 다양한 구분자 (Delimiters)
- 공백, 언더바(_), 슬래시(/) 혼용
- 예: "1k 0402 5%", "4.7k/0603/1%", "2.2M_1206_0.1%"

### 3. 다양한 저항값 표기 (Notation)
- 소수점 표기: 4R7, 1k5, 2R2, 0R47
- 예: "4R7 0603 5%", "1k5 0805 1%"

### 4. 엣지 케이스 (Edge Cases)
- 극단값: 1Ω, 10MΩ, 0.1Ω
- 옵셋 패키지: 01005, 0201
- 옵셋 오차: ±0.1%, ±0.25%

## 출력 형식

`tests/test-cases.json`:

```json
{
  "timestamp": "2026-03-08T14:00:00.000Z",
  "seed": 12345,
  "categories": [
    {
      "id": "structured",
      "label": "구조적 표준 입력",
      "cases": [
        { "input": "1k 0402 5%", "expected_resistance": 1000, "expected_package": "0402", "expected_tolerance": 5 },
        ...
      ]
    },
    ...
  ]
}
```

## GLM API 호출 방법

GLM API(`glm-4.7-flash`)를 사용하여 다음 프롬프트로 테스트 케이스 생성:

```
전자 부품 테스트 전문가입니다.
저항값 파싱 테스트를 위한 다양한 입력 케이스를 20개 생성해서 JSON으로 반환하세요.

카테고리별 5개씩:
1. 구조적 표준 입력: 공백/슬래시/언더바 구분자, 저항값 1k/4.7k/10K/2.2M/100R/4R7/1k5/2R2, 패키지 0201/0402/0603/0805/1206, 오차 0.1%/1%/5%
2. 다양한 구분자: 공백/언더바/슬래시 혼용
3. 다양한 저항값 표기: 4R7/1k5/2R2/0R47 소수점 표기
4. 엣지 케이스: 극단값 1Ω/10MΩ/0.1Ω, 옵셋 패키지 01005/0201, 옵셋 오차 ±0.1%/±0.25%

JSON 형식:
{
  "categories": [
    {
      "id": "structured",
      "label": "구조적 표준 입력",
      "cases": [
        {"input": "1k 0402 5%", "resistance": 1000, "package": "0402", "tolerance": 5},
        ...
      ]
    },
    ...
  ]
}

설명이나 텍스트 없이 JSON만 반환하세요.
```

## 사용자 요청 시

사용자가 `/test-cases-generator`를 입력하면:
1. 현재 GLM API 키 확인 (`.env`에서 `GLM_API_KEY`)
2. GLM API 호출하여 20개 테스트 케이스 생성
3. `tests/test-cases.json` 파일 생성
4. 생성된 파일 경로와 요약 정보 출력

## 랜덤 시드

매번 다른 케이스를 생성하기 위해 랜덤 시드를 JSON에 포함:
- 현재 시간 (milliseconds)를 시드로 사용
- 재현 가능성 확보
