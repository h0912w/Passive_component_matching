# Skill: 저항값 파싱 (Resistor Value Parsing)

## 개요
회로도에서 사용하는 저항 Value 표기를 파싱하여 저항값(Ω), 패키지 크기, 오차(%)를 추출하는 스킬.

## 입력 형식
사용자는 다양한 구분자와 순서로 저항 값을 표기한다:
```
1k 1005 5%          # 공백 구분
10k/0603/1%         # 슬래시 구분
4.7k_1005_5%        # 언더바 구분
100R	0402	1%      # 탭 구분
0805 2.2M 5%        # 순서 다름 (패키지가 먼저)
5% 1k 1005          # 순서 다름 (오차가 먼저)
```

## 파싱 규칙

### 1. 토큰 분리
구분자: 공백 ` `, 슬래시 `/`, 언더바 `_`, 탭 `\t`
```javascript
const tokens = input.trim().split(/[\s\/\_\t]+/);
```

### 2. 저항값 감지 (단위 R, k, K, M 포함)
정규식: `/^(\d+\.?\d*)(R|r|k|K|M|m)$|^(\d+\.?\d*)(R|r|k|K|M|m)(\d+)$/`

변환 규칙:
| 표기 | 의미 | 값(Ω) |
|------|------|-------|
| `100R` | 100 Ohm | 100 |
| `4R7` | 4.7 Ohm | 4.7 |
| `1k` | 1 kOhm | 1000 |
| `1k5` | 1.5 kOhm | 1500 |
| `4.7k` | 4.7 kOhm | 4700 |
| `10k` | 10 kOhm | 10000 |
| `2.2M` | 2.2 MOhm | 2200000 |
| `1M` | 1 MOhm | 1000000 |

- `R` = ×1 (또는 소수점 역할)
- `k`/`K` = ×1000
- `M` = ×1000000

### 3. 패키지 감지 (사전 정의 목록 매칭)
토큰이 아래 목록에 포함되면 패키지로 판정:

**Metric 패키지**:
`0402`, `0603`, `1005`, `1608`, `2012`, `2512`, `3216`, `3225`, `4516`, `4532`, `5025`, `6332`

**Imperial 패키지**:
`01005`, `0201`, `0402`, `0603`, `0805`, `1206`, `1210`, `1806`, `1812`, `2010`, `2512`

> **주의**: `0402`, `0603`, `2512`는 metric과 imperial 양쪽에 존재.
> 모호한 경우 기본값은 **metric**으로 간주하되, 사용자 설정으로 변경 가능.

### 4. 오차 감지 (% 포함)
정규식: `/^(\d+\.?\d*)%$/`

예: `1%`, `5%`, `0.1%`, `0.5%`, `10%`, `20%`

### 5. 순서 무관 처리
3개 토큰 각각에 위 규칙을 적용하여 타입(저항값/패키지/오차)을 판별.
순서에 의존하지 않으므로 어떤 순서로 입력해도 동작.

## 출력 형식
```json
{
  "resistance_ohms": 1000,
  "resistance_display": "1kΩ",
  "package_input": "1005",
  "package_metric": "1005",
  "package_imperial": "0402",
  "tolerance_percent": 5,
  "original_input": "1k 1005 5%",
  "parse_success": true,
  "error_message": null
}
```

## 에러 케이스
- 토큰이 3개가 아닌 경우 → 부분 파싱 시도, 누락 필드는 null
- 저항값 단위 없는 순수 숫자 → 저항값으로 추정 (패키지 목록에 없는 경우)
- 인식 불가 토큰 → `error_message`에 기록
